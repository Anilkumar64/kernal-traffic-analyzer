## WHAT IS BUILT AND WORKING

### ✅ PHASE 1 — Kernel correctness
- `skb_to_full_sk()` replacing raw `skb->sk`
- `sock_i_uid()` for kernel-authoritative UID
- 4-layer resolution pipeline: sock cache → inode cache → flow cache → async workqueue
- `put_pid()` + RCU lock fixes (original bugs fixed)
- PID liveness check before attribution
- All memory leaks fixed on rmmod

### ✅ PHASE 2 — Connection model
- Canonical 5-tuple key (lower IP always src — forward+reverse merge)
- Full TCP state machine: SYN_SENT → SYN_RECV → ESTABLISHED → FIN_WAIT → CLOSED
- Live bandwidth rate (1-second sliding window per connection)
- Per-process aggregation (`/proc/traffic_analyzer_procs`)
- DNS noise flagged and separated (`/proc/traffic_analyzer_dns`)

### ✅ PHASE 3 — Process intelligence
- Full executable path via `task->mm->exe_file` + `d_path()` (async, workqueue)
- Anomaly detection: CONN_BURST, PORT_SCAN, HIGH_CONNS, SYN_FLOOD, HIGH_BW
- Connection velocity tracker per process (new conns/sec, unique ports/sec)
- Top-5 connections per process by bytes
- Protocol distribution % (TCP_PCT, UDP_PCT)
- `/proc/traffic_analyzer_anomalies`

### ✅ PHASE 4 — DNS resolution (IP → domain)
- Full RFC 1035 DNS wire-format parser in kernel
- Intercepts DNS response packets from port 53
- Extracts A records, stores IP→domain in hash table with TTL
- `domain[]` field in every `traffic_entry`
- `/proc/traffic_analyzer_dns_map`
- DOMAIN column in connections view

### ✅ PHASE 5 — Network path (route store kernel side)
- `route_store.c`: hash table keyed by remote IP
- Stores up to 32 hops per IP: IP, hostname, RTT, city, country, lat/lon, ASN, org
- Status machine: PENDING → RUNNING → DONE / FAILED
- Auto-requests traceroute when TCP connection reaches ESTABLISHED
- Skips private/loopback/multicast IPs (is_routable() filter)
- `/proc/traffic_analyzer_routes` (R/W — daemon writes hop data here)
- `/proc/traffic_analyzer_routes_pending` (R — daemon polls for work)
- Write protocol for daemon: DEST / STATUS / HOP lines

## PROC FILE CONTRACT (FINAL — DO NOT CHANGE)

### `/proc/traffic_analyzer` — connections
```
PID|UID|PROCESS|RESOLVED|STATE|DNS|PROTO|
SRC_IP|DEST_IP|SRC_PORT|DEST_PORT|DOMAIN|
OUT_BYTES|IN_BYTES|OUT_PKTS|IN_PKTS|
RATE_OUT_BPS|RATE_IN_BPS|
FIRST_SEEN|LAST_SEEN|DURATION|CLOSED_AT
```
Column indices (0-based): 0=PID 1=UID 2=PROCESS 3=RESOLVED 4=STATE
5=DNS 6=PROTO 7=SRC_IP 8=DEST_IP 9=SRC_PORT 10=DEST_PORT 11=DOMAIN
12=OUT_BYTES 13=IN_BYTES 14=OUT_PKTS 15=IN_PKTS
16=RATE_OUT_BPS 17=RATE_IN_BPS
18=FIRST_SEEN 19=LAST_SEEN 20=DURATION 21=CLOSED_AT

### `/proc/traffic_analyzer_procs` — per-process
```
PID|UID|PROCESS|EXE|
TCP_CONNS|UDP_CONNS|TOTAL_CONNS|SYN_PENDING|
OUT_BYTES|IN_BYTES|OUT_PKTS|IN_PKTS|
RATE_OUT_BPS|RATE_IN_BPS|
TCP_PCT|UDP_PCT|ANOMALY|
TOP1|TOP2|TOP3|TOP4|TOP5
```
TOP format: `domain_or_ip:port/proto/bytes`

### `/proc/traffic_analyzer_dns` — DNS flows
```
PID|UID|PROCESS|SRC_IP|DEST_IP|SRC_PORT|DEST_PORT|
OUT_BYTES|IN_BYTES|OUT_PKTS|IN_PKTS|LAST_SEEN_AGO
```

### `/proc/traffic_analyzer_anomalies` — anomalies only
```
PID|UID|PROCESS|EXE|ANOMALY|
NEW_CONNS_LAST_SEC|UNIQUE_PORTS_LAST_SEC|
TOTAL_CONNS|SYN_PENDING|RATE_OUT_BPS|RATE_IN_BPS
```

### `/proc/traffic_analyzer_dns_map` — IP→domain map
```
DOMAIN|IP|TTL_REMAINING|QUERIED_BY_PID|QUERIED_BY_COMM|FIRST_SEEN|LAST_SEEN
```

### `/proc/traffic_analyzer_routes` — route hops (READ)
```
DEST_IP|DOMAIN|STATUS|TOTAL_HOPS|HOP_N|HOP_IP|HOST|RTT_MS|
CITY|COUNTRY|CC|LAT_E6|LON_E6|ASN|ORG
```

### `/proc/traffic_analyzer_routes_pending` — daemon work queue (READ)
```
<ip> <domain_or_dash>      (one line per IP needing traceroute)
```

### Write commands (all proc files accept these):
```
echo "clear"           → wipe all traffic stats
echo "clear_dns"       → wipe DNS flow entries
echo "clear_closed"    → wipe CLOSED connection entries
echo "clear_dns_map"   → wipe IP→domain mappings
echo "clear_routes"    → wipe all route data
DEST/STATUS/HOP lines  → daemon submits traceroute results
```

## KERNEL CODING RULES (learned from bugs fixed)

1. `DEFINE_HASHTABLE` never with `static` keyword — causes parse error
2. `DECLARE_HASHTABLE` in headers causes multiple-definition — use `extern struct hlist_head name[1 << BITS]`
3. Stack frames must stay under 1024 bytes — large arrays go module-level static
4. `get_mm_exe_file()` removed in kernel 6.7 — use `rcu_dereference(mm->exe_file)` + `get_file()`
5. `fput()` needs `<linux/file.h>` not just `<linux/fs.h>`
6. `ip_hdrlen()` needs `<net/ip.h>` not just `<linux/ip.h>`
7. `IPPROTO_TCP/UDP` need `<linux/in.h>` in every .c that uses them
8. `struct hlist_node` needs `<linux/list.h>` in every header that embeds it
9. Always `put_pid()` after every `find_get_pid()` — leaked one per packet before
10. `pid_task()` requires `rcu_read_lock()` — use-after-free without it
11. `spin_lock_bh()` not `spin_lock()` in code called from softirq context
12. `for_each_process()` requires `rcu_read_lock()`
13. `files->file_lock` spinlock before accessing `fdtable`
